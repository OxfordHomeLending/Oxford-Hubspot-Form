# Oxford Home Lending lead form

A dynamic, branching mortgage lead-capture form. React + Vite, designed to be
hosted on GitHub Pages and embedded in Webflow as an iframe. On submit it posts
one clean JSON payload to an n8n webhook, which fans out to Velocify (speed to
lead) and HubSpot.

Brand and palette come from the website design tokens: Red Main `#C42D2D`,
Oxford Navy `#021D38`, Oxford Blue `#2964D8`, and the Oxford gray scale.

---

## The flow (completion-optimized)

The order is intentional: open with one easy tap, grab the first name early to
personalize the rest, build momentum with low-friction taps, and defer the
higher-commitment asks (email, phone, consents) to the end once the person is
invested.

1. **Intent** - one tap: Buying / Refinancing / Accessing equity
2. **First name** - friendly, and powers every "name" mention afterward
   - branches on intent:
     - **Buying** -> state, occupancy, target price, stage, agent
     - **Refinance / Equity** -> property address, primary residence, loan amount
3. **Veteran -> employment -> income -> credit** (shared, easy taps)
4. **Contact details** (last name, email, phone) -> **preferred contact**
5. **Credit authorization** -> **TCPA consent** -> submit

Copy is dynamic throughout (`title`/`sub`/`note` can be functions of the answers
object), so the form greets the client by name, echoes their chosen state, and
shows reassurance exactly where anxiety spikes (credit question, contact step).

## Configuration

Copy `.env.example` to `.env`:

- `VITE_N8N_WEBHOOK_URL` - your n8n webhook. Defaults to the known endpoint.
- `VITE_REDIRECT_URL` - optional; redirect here on success instead of the inline thank-you.
- `VITE_FORM_TOKEN` - optional shared secret, sent as `X-Oxford-Form-Token`.

## Deploy + embed in Webflow

The form is meant to live in an **iframe** so it is fully isolated from Webflow's
CSS and cannot be broken by site styles.

1. **Build**: `npm run build` produces `/dist` with relative asset paths
   (`base: './'` in `vite.config.js`), so it works from any subpath.
2. **Host on GitHub Pages**: push the repo, then either commit `/dist` to a
   `gh-pages` branch or use a GitHub Action to build and publish. Pages serves
   over HTTPS at `https://YOURNAME.github.io/REPO/`.
3. **Embed**: in Webflow, drop an **Embed** element and paste `embed-webflow.html`,
   replacing `YOURNAME/REPO` with your Pages URL. The included script listens for
   the form's height messages and resizes the iframe, so there is no inner
   scrollbar.

### One hard requirement: the webhook must be HTTPS

GitHub Pages (and your Webflow site) are HTTPS. A browser will **block** a POST
from an HTTPS page to an `http://` webhook as mixed content. Before going live,
put n8n behind TLS:

- a reverse proxy (Caddy or Nginx + Let's Encrypt) in front of `:5678`, or
- a Cloudflare Tunnel to the droplet.

Then set `VITE_N8N_WEBHOOK_URL` to the `https://` URL. Also set **CORS** in the
n8n Webhook node Options ("Allowed Origins") to your Webflow domain, or the
browser will block the response.

## Security and privacy

What this form guarantees, and what depends on your infrastructure. (Being
precise here on purpose; do not over-promise to compliance.)

**In transit**
- When the form page is HTTPS and the webhook is HTTPS, the entire JSON payload
  is encrypted end to end by TLS. There is no third hop: the browser posts
  straight to your n8n endpoint.

**At rest, in the browser**
- The only thing persisted on the device is a first-touch attribution cookie
  (`ohl_attr`): click IDs, UTMs, landing page, referrer, and a random UUID.
  **It never stores name, email, or phone.** PII exists only in memory for the
  moment of submission.
- The cookie is set `SameSite=Lax`, and `Secure` is added automatically on HTTPS.
- No `localStorage`/`sessionStorage` is used.

**No PII in logs**
- All debug logging is gated behind `import.meta.env.DEV`, so a production build
  logs nothing to the console.

**Abuse protection (built in)**
- **Honeypot**: a hidden `company` field that humans never see. If a bot fills
  it, the form shows the success screen and **silently drops the submission with
  no webhook hit**.
- **Timing trap**: completions faster than 2.5s are flagged `suspectedBot` in the
  payload (`meta.suspectedBot`) so your n8n flow can review or drop them.
- Optional `X-Oxford-Form-Token` header for a server-side filter in n8n.

**At rest, downstream**
- Encryption of the lead once it reaches n8n, Velocify, and HubSpot is the
  responsibility of those systems and your account configuration. This form's
  job ends at delivering the payload securely.

## Testing

Before you test live, the harness in `test/run-tests.mjs` imports the **real**
`src/flow.js` and `src/tracking.js`, walks each of the three branch paths exactly
the way the form does, builds the payload with the real `buildPayload()`, and
asserts the JSON is correct (right fields populated per path, other path's fields
empty, consents granted with timestamp + text, full attribution captured, stable
`leadUuid`). It also unit-tests the honeypot and timing logic.

```bash
npm test               # prints each path's JSON + a PASS/FAIL summary
npm run test:samples   # same, and rewrites sample-payloads.json
```

All 134 checks pass on the current build.

## Payload shape

```jsonc
{
  "leadUuid": "uuid",            // idempotency key for dedupe in n8n
  "submittedAt": "ISO",
  "source": "oxford-web-form",
  "formVersion": "2.0.0",
  "contact":  { "firstName", "lastName", "email", "phone", "preferredContact" },
  "loan":     { "intent", "veteran",
                "purchaseState", "willOccupy", "purchasePrice", "purchaseProcess", "workingWithRealtor",
                "currentAddress", "primaryResidence", "estimatedLoanAmount",
                "employmentStatus", "householdIncome", "creditCategory" },
  "consents": { "creditAuthorization": { "granted", "timestamp", "text" },
                "tcpa":               { "granted", "timestamp", "text" } },
  "meta":     { "formStartedAt", "elapsedMs", "honeypotTriggered", "suspectedBot" },
  "tracking": { "gclid", "fbclid", "msclkid", ... , all UTMs,
                "gaClientId", "hubspotutk", "leadUuid",
                "referrer", "landingPage", "submitPage",
                "userAgent", "language", "timezone", "screen", "viewport" }
}
```

`contact` + `loan` map to Velocify LeadAdd; `consents` to the TCPA/credit fields;
`tracking` to Velocify hidden fields + HubSpot properties; `meta` to spam routing.

Purchase-path leads carry the purchase fields and a null `currentAddress`;
refinance/equity leads carry `currentAddress` + loan amount and leave the
purchase fields empty. This is verified by the test harness.
